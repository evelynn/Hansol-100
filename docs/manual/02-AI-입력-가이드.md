# 02. AI로 손쉽게 입력하기 (권장)

이 시스템의 가장 큰 장점은 **업무를 말이나 기존 문서로 설명하면 AI가 표준 보드를
자동으로 만들어 준다**는 점입니다. 담당자는 JSON 문법을 몰라도 되고, 익숙한 방식
(설명, 규정, 업무분장표, 회의록)으로 입력하면 됩니다.

이 시스템은 **Claude Code용 에이전트 스킬**로 만들어져 있어, 저장소 안에서
Claude Code를 실행하면 스킬(`SKILL.md`)이 자동으로 인식됩니다.

---

## 1. 무엇이 가능한가 (개요)

```
[담당자]  "구매 요청서를 올리면 부서장이 승인하고, 반려되면 보완해서
          다시 올려요. 승인되면 구매팀이 발주하고 재무팀이 정산합니다."
                              │
                              ▼
[AI]  ① 부서(레인)·단계·업무·연결을 자동 식별
      ② board-v1 JSON 자동 작성 (profile: hansol)
      ③ render 로 SVG/PNG 그리기
      ④ audit 로 가독성 자가 점검, 필요하면 스스로 수정
      ⑤ 부족한 정보는 담당자에게 한두 가지만 되물음
                              │
                              ▼
[산출물]  표준 보드 이미지 + JSON + 애니메이션
```

AI가 담당자 대신 [`01-업무프로세스-입력-매뉴얼`](01-업무프로세스-입력-매뉴얼.md)의
전 과정을 수행합니다.

---

## 2. 시작하기 (3단계)

1. **저장소를 열고 Claude Code 실행**

   ```bash
   git clone https://github.com/evelynn/Hansol-100.git
   cd Hansol-100 && npm install
   claude          # Claude Code 실행 (또는 사내에 배포된 방식대로)
   ```

   스킬 디렉터리(`~/.claude/skills/`)에 설치해 두면 어느 폴더에서든 스킬을
   부를 수 있습니다(README의 설치 참고).

2. **업무를 설명한다** — 아래 프롬프트 틀을 참고해 채팅으로 입력.

3. **결과를 확인하고 다듬는다** — AI가 만든 보드를 보고, 고칠 점을 말로 지시.

---

## 3. 프롬프트 틀 (복사해서 사용)

### 3.1 기본 틀

```
한솔 업무 프로세스 보드로 아래 업무를 그려줘. profile은 hansol로 하고,
반려·재작업 경로가 있으면 loop로, 자주 지연되는 단계는 bottleneck으로 표시해줘.
다 그리면 audit도 돌려서 nodePiercings가 0인지 확인해줘.

[업무 설명]
- 프로세스 이름:
- 관여 부서(순서대로):
- 단계(순서대로):
- 진행 내용(누가 무엇을 하는지, 반려/예외 포함):
- 근거 규정(있으면):
```

### 3.2 아주 간단하게

정보를 다 정리하지 않아도 됩니다. 이렇게만 말해도 AI가 나머지를 되물어 채웁니다.

```
우리 팀 구매 요청 프로세스를 hansol 프로파일 보드로 만들어줘.
요청부서가 올리면 부서장 승인 → 구매팀 발주 → 재무팀 정산이고,
부서장이 반려하면 보완해서 다시 올려.
```

### 3.3 기존 문서로부터 변환

문서를 붙여넣거나 파일 경로를 주면 됩니다.

```
첨부한 '구매관리규정'과 '업무분장표'를 읽고, 구매 프로세스를 hansol 프로파일
보드로 만들어줘. 조문은 각 업무의 근거(refs)로 달아줘.
```

변환에 유용한 사내 문서 예시:
- **업무분장표 / R&R** → 부서(레인)와 담당 업무를 뽑기 좋음
- **사내 규정·지침·SOP** → 단계와 근거(refs)를 뽑기 좋음
- **전결규정** → 어떤 단계가 `key`(핵심 승인)인지 판단
- **회의록 / 업무 인수인계 문서** → 실제 흐름과 반려·예외 파악
- **기존 순서도(PPT/이미지 설명)** → 기존 그림을 표준 보드로 재작성

---

## 4. 예시 대화

**담당자:**
> 신규 협력업체 등록 프로세스를 hansol 보드로 만들어줘. 현업이 등록 요청하면
> 구매팀이 서류를 검토하고, 미비하면 반려해. 이상 없으면 품질팀이 실사하고,
> 재무팀이 신용도를 확인한 뒤 구매팀장이 최종 승인해. 실사가 자주 지연돼.

**AI(요약 동작):**
> 부서 4개(현업/구매팀/품질팀/재무팀), 단계 4개(요청/서류검토/실사·심사/승인)로
> 잡았습니다. '협력업체 실사'는 지연이 잦다고 하셔서 `bottleneck`(지연),
> 최종 승인은 `key`(핵심), 서류 미비 반려는 `loop`(반려)로 표시했습니다.
> board-v1 JSON을 만들고 렌더링했습니다 — audit 결과 nodePiercings 0, score 7.x.
> 확인해 보시고 수정할 부분을 알려주세요.
>
> 한 가지 확인: 재무팀 신용도 확인은 품질 실사와 **동시**인가요, **이후**인가요?

**담당자:**
> 동시야. 그리고 최종 승인 근거로 '협력업체관리규정 제7조' 달아줘.

**AI:** (수정 후 다시 렌더·audit)

이처럼 **한 번에 완벽할 필요가 없습니다.** 말로 고쳐 나가면 됩니다.

---

## 5. 좋은 입력을 위한 팁

- **부서를 업무가 넘어가는 순서로** 나열하면 배치가 깔끔합니다(요청자 먼저,
  최종 결재자 나중).
- **반려·재작업·예외를 꼭 언급**하세요. AI가 `loop`로 그립니다. 빠지면 실제 절차와
  달라집니다.
- **병목(자주 지연되는 단계)을 알려주세요.** `bottleneck`으로 강조돼 개선 포인트가
  드러납니다.
- **근거가 중요하면 규정·조문을 함께** 주세요. 카드에 `근거`로 인용됩니다.
- 강조는 **핵심만** — "전부 다 중요"라고 하면 강조가 무의미해집니다. AI에게
  "단계당 핵심 1~2개만"이라고 일러도 됩니다.
- 이름·라벨은 **짧은 동사구**가 좋습니다("부서장 검토·승인").

---

## 6. AI 결과물, 이렇게 확인하세요

AI가 만든 보드를 그대로 쓰기 전에 담당자가 사실관계를 검토해야 합니다.

- [ ] **부서·단계가 실제와 맞는가** — 순서·명칭
- [ ] **반려/예외 경로가 반영됐는가** — `loop`(반려)로 표시됐는지
- [ ] **핵심 승인 단계가 `key`인가** — 전결 권한 기준
- [ ] **근거 규정이 정확한가** — 조문 번호는 사람이 검증
- [ ] **누락된 업무는 없는가**
- [ ] `audit`의 **nodePiercings가 0**, `validate --strict` 통과

> ⚠️ AI는 초안 작성을 돕는 도구입니다. **규정 조문·전결 권한·법적 근거는 반드시
> 담당자가 확인**하세요. 실제 결재·전결 권한은 사내 규정 및 전결규정을 따릅니다.

---

## 7. 여러 프로세스를 한꺼번에

부서 업무를 대량으로 정리할 때는 이렇게 지시할 수 있습니다.

```
아래 목록의 업무들을 각각 hansol 프로파일 보드로 만들어서 fixtures/ 아래에
개별 JSON으로 저장하고, 각각 render + audit 해줘. 하나라도 nodePiercings가
0이 아니면 고쳐줘.

1) 구매 요청·발주
2) 신규 협력업체 등록
3) 품질 이슈(클레임) 처리
4) 연차·휴가 결재
```

AI가 순서대로 작성·렌더·점검하고, 문제가 있으면 스스로 수정합니다.

---

## 8. 자동화(선택) — 반복 작업 줄이기

- **표준 검증을 CI에 걸기:** 저장소에 보드 JSON을 커밋하면, CI에서
  `validate --strict`로 모든 보드의 품질을 자동 게이트할 수 있습니다
  (`.github/workflows/ci.yml` 참고).
- **세션 시작 훅:** Claude Code on the web에서 매 세션 자동으로 `npm install`·
  테스트가 돌도록 SessionStart 훅을 둘 수 있습니다.
- **일괄 재렌더:** 규정이 개정되면 관련 JSON만 수정해 커밋 → 이미지 일괄 재생성.

---

## 요약

1. 저장소에서 Claude Code를 연다.
2. 업무를 말이나 문서로 설명한다 (3장 프롬프트 틀).
3. AI가 표준 보드(JSON) + 이미지 + 애니메이션을 만들고 스스로 품질 점검한다.
4. 담당자는 사실관계(부서·단계·반려·근거)를 검토·수정 지시한다.
