# 00. 한솔 업무 프로세스 보드 — 소개

## 이 시스템은 무엇인가요?

**한솔 업무 프로세스 보드**는 사내 업무 절차를 누구나 같은 방식으로 그리고,
같은 기준으로 검증할 수 있게 해 주는 시스템입니다. 구매 요청, 결재·승인,
채용, 품질 이슈 처리처럼 "누가 · 언제 · 무엇을 하는가"가 있는 모든 업무를
**하나의 표준 그림(스윔레인 보드)** 으로 만들 수 있습니다.

- **가로(레인)** = 부서·담당 (예: 요청부서 → 부서장 → 구매팀 → 재무팀)
- **세로(단계)** = 업무가 지나가는 순서 있는 국면 (예: 요청 → 승인 → 발주 → 검수)
- **카드(업무)** = 특정 부서가 특정 단계에서 하는 일 하나
- **화살표(연결)** = 업무의 흐름 · 정보 공유 · 반려/재작업

![구매 프로세스 예시](../../assets/example-hansol.png)

## 왜 쓰나요? (도입 배경)

기존에는 부서마다 파워포인트·엑셀·손그림으로 제각각 업무 흐름을 그렸습니다.
표현 방식이 달라 **비교·인수인계·감사·개선이 어렵고**, 그림이 오래되면 실제
절차와 어긋납니다. 이 시스템은 다음을 해결합니다.

| 문제 | 이 시스템의 해결 |
|---|---|
| 부서마다 그림 형식이 다름 | 모든 보드가 하나의 표준(`board-v1`)을 따름 |
| 그림이 지저분하고 선이 꼬임 | `audit`가 선 겹침·꼬임을 **자동 점검·점수화** |
| 만드는 데 시간이 오래 걸림 | **AI가 자연어 설명을 보드로 자동 변환** |
| 버전 관리가 안 됨 | JSON 텍스트라 Git 등으로 이력·변경 추적 가능 |
| 근거(규정·전결) 표시가 빠짐 | 카드마다 `근거`(사규·지침) 인용 가능 |

## 두 가지 입력 방법

이 시스템에 업무 프로세스를 넣는 방법은 두 가지입니다. **대부분의 담당자는
AI 입력만 알아도 충분합니다.**

1. **AI로 입력 (권장)** — 업무를 말로 설명하거나 기존 문서(규정, 업무분장표,
   회의록)를 붙여넣으면 AI(Claude Code)가 표준 보드로 변환하고 그림까지 그려
   줍니다. JSON 문법을 몰라도 됩니다.
   → [`02-AI-입력-가이드.md`](02-AI-입력-가이드.md)

2. **직접 입력** — 표준 양식(JSON)을 손으로 작성합니다. 세밀하게 다듬거나
   AI 결과물을 수정할 때 유용합니다.
   → [`01-업무프로세스-입력-매뉴얼.md`](01-업무프로세스-입력-매뉴얼.md)

## 산출물

하나의 프로세스 입력에서 다음 세 가지가 나옵니다.

- **보드 이미지** — SVG(항상), PNG(선택). 보고서·게시·인쇄에 사용.
- **품질 점검 리포트** — 가독성 점수와 문제 지점.
- **리빌 애니메이션** — 단계 순서대로 나타나는 SVG. 교육·발표용.
